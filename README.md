Network Traffic Analyzer & Monitoring System

A real-time network traffic analyzer with a Tkinter GUI dashboard. It captures live packets, classifies them by protocol, visualizes traffic through charts, detects suspicious activity, and exports analysis reports.

Features


Dashboard — live stat cards (packets, packets/sec, data volume, active protocols, alerts, top protocol), a protocol pie chart, a per-protocol bar chart, live throughput graphs, and a rolling table of recent packets.
Live Packets — full packet table with protocol filters, free-text search, sortable columns, and a per-packet detail pane.
Charts — full-size protocol pie chart, bar chart, and live packets/sec & bytes/sec throughput graphs.
Alerts — a live log of detected anomalies: high-traffic sources, ICMP floods, and traffic to known suspicious ports.
Summary — session statistics, a protocol breakdown table, and two export options:

Export CSV — raw packet dump
Full Report (CSV + JSON) — traffic summary + anomaly findings, generated by the analysis engine





Tech Stack


Python 3.9+
Tkinter — GUI (bundled with most Python installers)
Matplotlib — charts
Scapy — live packet capture
Npcap (Windows only) — required by Scapy for raw packet capture


Requirements


Administrator/root privileges (raw packet capture requires elevated access on every OS)
Windows: Npcap installed with "Install Npcap in WinPcap API-compatible Mode" checked
macOS/Linux: run with sudo


Setup


Clone the repo and cd into it.
Install dependencies:


bash   pip install -r requirements.txt
   pip install scapy


Windows: install Npcap (see Requirements above).
Run with elevated privileges:

Windows: open an Administrator terminal
macOS/Linux: prefix the run command with sudo





Running the App

bashpython main.py

This opens the main window with a header bar (Start / Stop / Clear), a five-tab navigation bar, and a live clock. Click START to begin capturing.

Integration & QA Notes

The capture, analysis, and GUI modules were originally built independently and were not fully connected at runtime — most notably, the anomaly-detection engine in analyzer.py never ran during live capture, so the Alerts page always showed zero findings regardless of actual traffic.

This was fixed by connecting analyzer.py's detection functions into main.py's refresh loop (running every ~3 seconds) so genuine findings populate the Alerts page and color-code matching packets, and by wiring a Full Report button into the Summary tab that calls analyzer.py's CSV/JSON report generator directly.

Verified with synthetic traffic (a simulated port scan, ICMP flood, and high-traffic source): all 7 automated static tests pass, and all 3 injected attack patterns were correctly detected with no duplicate alert spam across repeated refresh cycles.
